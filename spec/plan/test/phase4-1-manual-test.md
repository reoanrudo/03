# Phase 4-1: 難易度フィルタリングのテスト

## 注意: Playwright MCP サーバーが使用できません

Playwright ブラウザのインストールには `sudo` 権限が必要で、現在の環境からは実行できません。

## 手動テストガイド

以下の手順で手動テストを行ってください。

### 1. 開発サーバーを起動済み確認

開発サーバーは既に起動しています：
```
http://localhost:3000
```

ブラウザでこのURLにアクセスしてください。

### 2. 「全て」フィルターを確認

期待される結果：
- [ ] 「全て」ボタンが選択状態（灰色/オレンジ）
- [ ] 全ての曲（3曲）が表示されていること
  - きらきら星（初級）
  - Wonder Wall（中級）
  - Stairway to Heaven（上級）

### 3. 「初級」フィルターを確認

操作：
1. 「初級」ボタンをクリック

期待される結果：
- [ ] 初級曲のみが表示されていること
  - きらきら星（初級）のみ
- [ ] ボタンの色が緑色（`bg-green-500`）であること
- [ ] 他の曲が非表示であること

### 4. 「中級」フィルターを確認

操作：
1. 「中級」ボタンをクリック

期待される結果：
- [ ] 中級曲のみが表示されていること
  - Wonder Wall（中級）のみ
- [ ] ボタンの色が黄色（`bg-yellow-500`）であること
- [ ] 他の曲が非表示であること

### 5. 「上級」フィルターを確認

操作：
1. 「上級」ボタンをクリック

期待される結果：
- [ ] 上級曲のみが表示されていること
  - Stairway to Heaven（上級）のみ
- [ ] ボタンの色が赤色（`bg-red-500`）であること
- [ ] 他の曲が非表示であること

### 6. 「全て」に戻る

操作：
1. 「全て」ボタンをクリック

期待される結果：
- [ ] 全ての曲が再表示されていること
  - きらきら星（初級）
  - Wonder Wall（中級）
  - Stairway to Heaven（上級）

### 7. トランジション確認

期待される結果：
- [ ] フィルター変更時のアニメーション（`scale-110`）がスムーズであること
- [ ] ボタンのホバー効果（`hover:scale-105`）が機能すること
- [ ] アクティブなフィルターボタンに影（`shadow-lg`）があること

### 8. モバイル対応（レスポンシブ）

操作：
1. ブラウザウィンドウサイズを変更（スマートフォンサイズなど）

期待される結果：
- [ ] レスポンシブデザインが正しく機能すること
  - `md:grid-cols-2` - タブレットサイズ（768px以上）で2列表示
  - `lg:grid-cols-1` - ラージサイズ（1024px以上）で1列表示

### 9. 進捗表示

期待される結果：
- [ ] 既存の進捗がある曲には「ベストスコア」と「クリア回数」が表示されること
- [ ] 進捗のない曲には進捗情報が表示されないこと

## 開発サーバー停止

テスト完了後、開発サーバーを停止するには以下を実行してください：

```bash
pkill -f "vite"
```

または、ターミナルで `Ctrl+C` を押してください。
